**Status:** READY FOR EXECUTION | **Priority:** HIGH

## 1. Objective
Use the high-speed "Flash" resource to generate the non-complex skeletal code for the decoupled architecture. This phase focuses on Structure over Intelligence, creating the containers that will eventually house the 3.1 logic.

## 2. Component Specifications

### A. lx_state.py (The Ledger)
Decoupling Logic: This module must handle all persistence independently.
**Class:** `lx_StateStore`
**Interfaces:** 
* `get_active_profile()`: Pulls the current state from disk/ChromaDB.
* `apply_delta(delta: dict)`: Merges new intelligence into the persistent layer.
* `sync_vector()`: Triggers the ChromaDB update for Success Vectors.

### B. core.py (The Engine)
Stateless Loop: The main orchestrator that only knows how to process Cognates.
**Class:** `ServoCore`
**Workflow:**
1. Request state from `lx_StateStore`.
2. Identify the next Cognate in the registry.
3. Execute and capture the StateDelta.
4. Send delta back to the Ledger.

### C. lx_cognates.py (The Registry)
Polymorphic Base: An abstract Cognate class.
**Stubs:** Skeletal implementations for:
* `lx_Observe`: The sensor array.
* `lx_Reason`: The planning node.
* `lx_Act`: The execution circuit.
* `lx_Integrate`: The memory processor.

## 3. Verification Gate (Phase B)
After the scaffold is generated, the `/benchmark/` suite must be run again.

**Success Condition:** `lx_correctness` must pass a "Mock Handshake" between the new `core.py` and `lx_state.py`.
**Failure Condition:** Any "As an AI" boilerplate generated by Flash during the scaffolding must be caught by `lx_lexicon` and purged.

---
**Audit Guardrails:**
- **State Desync**: `Cognate.execute` must return a **delta**, not a full state.
- **Boilerplate Noise**: No `if __name__ == "__main__":` blocks or prose leakage.
- **Registry Locking**: `step_key` must match registry keys exactly.
- **Method Locking**: `run_cycle` is the mandatory entry point. No `close_circuit` references.
