# Engineering Standards — LLM & Agentic Best Practices

**Version:** 1.0
**Status:** In-Effect

These standards are designed to minimize hallucinations and maximize the structural integrity of the code generated by local models

## 1. Code Generation Principles

**Atomic Commits / Edits**
*   Prefer making small, verified changes over sweeping refactors.
*   Each tool call should ideally address a single logical problem.

**Explicit Typing**
*   In Python, use type hints (`: str`, `-> list`) for all function signatures. This helps the model maintain "type context" when reading back its own code.

**Fenced Communication**
*   Always use language-specific fenced code blocks (````python`, ````json`, ````css`).
*   Never output raw code without a fence.

**Path Discipline**
*   Strictly use Project-Root-Relative paths. 
*   Always use forward slashes `/` even on Windows to prevent backslash-escape corruption in JSON payloads.

## 2. Language-Specific Guardrails

### Python
*   **Safety first:** Ensure all filesystem operations are confined to the sandbox.
*   **Error Handling:** Wrap external API calls (Ollama, ChromaDB) in try/except blocks to prevent loop crashes.

### UI / CSS
*   **Rich Aesthetics:** Use HSL colors, CSS variables, and modern typography (Google Fonts) as per the *Aesthetic Mandate*.
*   **Clean DOM:** Prefer semantic HTML over nested `div` soups.

## 3. Interaction Best Practices

**The "Context Load" Pattern**
*   Before modifying a complex file, always view the file to load its current state into the turn's context.

**Markdown Signaling**
*   Use GitHub-style Alerts (`> [!IMPORTANT]`, `> [!WARNING]`) in reports and plans to highlight critical architectural breakpoints.

**Truncation Awareness**
*   Respect the 16,000 character tool output cap. file read and youtube transcript tools support optional block argument, use block 1 after receiving truncation warning to get the next block. use targetd block argument to get specific block based on your episodic memory, or optimized tools.
